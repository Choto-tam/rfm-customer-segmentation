# rfm-customer-segmentation
# RFM-анализ клиентов интернет-магазина (Online Retail)
Данный проект представляет собой реализацию RFM-анализа (Recency, Frequency, Monetary) для сегментации клиентов на основе публичного датасета "Online Retail". RFM-анализ позволяет выделить группы клиентов по их покупательскому поведению и разработать персонализированные маркетинговые стратегии.

# Описание проекта
RFM-анализ — один из самых популярных методов сегментации клиентов в маркетинге. Он основан на трёх метриках:

* **Recency** (давность) — сколько дней прошло с последней покупки клиента.

* **Frequency** (частота) — общее количество покупок клиента.

* **Monetary** (денежная ценность) — общая сумма, потраченная клиентом.

В проекте выполняется:

* Загрузка и первичный анализ данных.

* Очистка данных: удаление пропусков, отрицательных значений, дубликатов.

* Расчёт метрик для каждого клиента.

* Присвоение баллов по каждой метрике (от 1 до 5) на основе квантилей.

* Формирование RFM-кода и выделение ключевых сегментов.

Результат — выделение групп клиентов с разной ценностью для бизнеса.

# Данные
Используется датасет **"Online Retail"** из UCI Machine Learning Repository. Это транзакционные данные интернет-магазина, зарегистрированного в Великобритании, за период с 01/12/2010 по 09/12/2011.

**Описание полей**:

* **InvoiceNo** — номер счёта-фактуры (уникальный идентификатор транзакции)

* **StockCode** — код товара

* **Description** — описание товара

* **Quantity** — количество единиц товара

* **InvoiceDate** — дата и время выставления счёта

* **UnitPrice** — цена за единицу товара (в фунтах стерлингов)

* **CustomerID** — уникальный идентификатор клиента

* **Country** — страна клиента

# Этапы анализа
## 1. Загрузка и первичный осмотр
* Импорт библиотек.

* Чтение Excel-файла.

* Просмотр первых строк, размерности, типов данных, количества пропусков.

* Выявление отрицательных количеств (возвраты).

## 2. Очистка данных
* Добавление поля **Revenue** = **Quantity** × **UnitPrice**.

* Удаление строк с пропущенным **CustomerID** (нельзя идентифицировать клиента).

* Преобразование **CustomerID** в целочисленный тип.

* Фильтрация: только положительные **Quantity** и **UnitPrice** (исключаем возвраты и нулевую/отрицательную цену).

* Удаление дубликатов строк.

## 3. Расчёт клиентских метрик
* Группировка по **CustomerID** и агрегация:

* Количество уникальных заказов (**nunique** по **InvoiceNo**) — **Frequency**.

* Сумма выручки (**sum** по **Revenue**) — **Monetary**.

* Общее количество купленных товаров (**sum** по **Quantity**).

* Минимальная и максимальная дата покупки (**min**, **max** по **InvoiceDate**).

## 4. Вычисление RFM-показателей
* Определение даты анализа как **max(InvoiceDate) + 1 день** (для корректного расчёта давности).

* **Recency** = разница в днях между датой анализа и последней покупкой клиента.

* **Frequency** = количество заказов.

* **Monetary** = общая выручка.

## 5. Присвоение RFM-баллов (квантильный метод)
* **R_score**: клиенты делятся на 5 групп по давности покупки. Самые «свежие» получают 5 баллов, самые «старые» — 1.

* **F_score**: деление на 5 групп по частоте. Самые частые — 5 баллов.

* **M_score**: деление на 5 групп по денежной ценности. Самые ценные — 5 баллов.

* Используется **pd.qcut** для равномерного распределения по группам.

## 6. Формирование RFM-кода и анализ сегментов
* Конкатенация трёх баллов в строку (например, "155", "511").

* Подсчёт количества клиентов в каждом сегменте.

* Вывод топ-10 наиболее часто встречающихся сегментов.


# Итоговые данные

В результате создаётся таблица ***results/rfm_table.csv***, содержащая для каждого клиента:

* **num_orders** — количество заказов

* **total_revenue** — общая выручка

* **total_items** — общее количество купленных товаров

* **first_purchase** — дата первой покупки

* **last_purchase** — дата последней покупки

* **recency** — давность (дней)

* **frequency** — частота

* **monetary** — денежная ценность

* **R_score**, **F_score**, **M_score** — баллы

* **RFM_score** — итоговый код сегмента

Примеры сегментов:

* **"555"** — лучшие клиенты: недавно покупали, часто и много тратят.

* **"111"** — потерянные клиенты: покупали давно, редко и мало.

* **"511"** — новые клиенты с большим потенциалом (недавно, но пока мало).

Анализ этих сегментов позволяет выработать маркетинговые рекомендации (удержание, повторные продажи, реактивация).

# Бизнес-ценность

RFM-анализ позволяет:

* увеличить удержание клиентов

* повысить эффективность маркетинга

* выделить ключевые сегменты

* оптимизировать работу с клиентской базой
